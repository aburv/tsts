## Types of Contributions

You can contribute to the project in several ways:

- **Documentation**: Improving, updating, or adding new documentation, guides, or examples.  
- **Bug Fixes**: Identifying and fixing issues in the codebase.  
- **Features/Enhancements**: Adding new functionality or improving existing features.  
- **Tests**: Adding or improving test coverage to ensure the codebase is reliable.  
- **Code Refactoring**: Improving code structure, readability, or performance without changing functionality.  
- **DevOps/CI Improvements**: Enhancing build, deployment, or automated check processes.  
- **Design/UX**: Improving the user interface, accessibility, or overall user experience.  

## How to Contribute

1. Fork the repository.  
2. **Create a new branch from `master`** for your feature or fix.  
3. Make your changes.  
4. **If your contribution includes code changes:**  
   - Ensure test coverage is **above 90%**.  
   - Fix all **lint issues**.  
   > Each pull request has automated checks, and **pull requests will not be reviewed until all checks pass**.
5. **Submit a pull request to `master`** and **assign a review request** to the appropriate reviewer(s).  
6. Once your pull request is approved and merged:  
   - Check out []() to see your contribution live on the **development environment**.  
   - The production environment goes live **once a month**.
